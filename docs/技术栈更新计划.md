# 技术栈更新计划

> 制定日期: 2026-06-22

## 一、现状分析

当前项目使用 Spring Boot 2.2.2 + Vue 2.6 技术栈，各核心依赖均为 2019-2020 年间版本，至今日已全面过时。项目共 132 个 Java 文件、约 12 个 Vue 页面组件，属于中小型单体应用。

### 关键问题

| 问题 | 影响 |
|---|---|
| Spring Boot 2.2.2 已停止维护 | 安全漏洞无补丁、无法使用现代 Spring 特性 |
| Vue 2 已于 2023-12-31 EOL | 不再获安全更新，生态已全面转向 Vue 3 |
| node-sass 已废弃 | 安装慢、内存占用高，新版 Node 无法安装 |
| FastJSON 1.2.8 | 存在多次高危 CVE（反序列化 RCE） |
| Shiro 1.3.2 | 已知 CVE-2020-1957 等漏洞 |
| 无自动化测试 | `pom.xml` 默认 `skipTests=true` |

## 二、升级路线（分两阶段）

### 阶段一：后端升级（Spring Boot 2.2 → 3.x）

**预计工作量**: 中等 | **风险**: 中

#### 1. 升级 JDK 1.8 → 17+

- 修改 `pom.xml` `<java.version>` 为 `17`
- Maven compiler source/target 改为 `17`
- 所有 `javax.*` 导入（34 处）需改为 `jakarta.*`：
  - `javax.servlet.http.HttpServletRequest` → `jakarta.servlet.http.HttpServletRequest`
  - `javax.validation.*` → `jakarta.validation.*`
- 涉及文件: `src/main/java/com/controller/CommonController.java`, `src/main/java/com/utils/ValidatorUtils.java` 等

#### 2. Spring Boot 2.2.2 → 3.2+ (LTS)

- 升级 `spring-boot-starter-parent` 至 `3.2.5`
- 升级 `mybatis-spring-boot-starter` 至 `3.0.3`
- 升级 `mybatis-plus` 至 `3.5.6`（同时移除 `mybatisplus-spring-boot-starter` 旧依赖）
- 升级 `shiro-spring` 至 `2.0.x`（注意 Shiro 2.0 包名从 `org.apache.shiro` → 仍为 `org.apache.shiro`，但内部 API 有较大变化）
- 升级 `hutool-all` 至 `5.8.28`
- 升级 `poi` 至 `5.2.5`
- 升级 `baidu-aip-sdk` 至 `4.32.0`
- FastJSON 1.2.8 → **替换为 Jackson**（Spring Boot 3 默认已集成），或升级为 FastJSON2 2.0.50
- `org.json.JSONObject` 替换为 Jackson `ObjectMapper`

#### 3. Shiro 安全框架升级

- Shiro 1.3.2 → Shiro Spring Boot Starter 2.0.x
- 需要确认项目中是否有自定义 Realm/Filter 配置（当前代码未发现显式 Shiro 配置类，可能通过注解 `@IgnoreAuth` 实现）
- 考虑迁移至 **Spring Security 6**（与 Spring Boot 3 深度集成），这是更长期的方案

#### 4. MyBatis Plus 配置调整

- `com.baomidou.mybatisplus.mapper.LogicSqlInjector` → `com.baomidou.mybatisplus.core.injector.LogicSqlInjector`
- `typeAliasesPackage` → `type-aliases-package`（Spring Boot 3 属性键命名规范变更）
- XML mapper 中的 `EntityWrapper` → `LambdaQueryWrapper`（`CommonController.java` 中有使用）

#### 5. 配置文件更新

`application.yml` 需修改：
```yaml
# 旧
mybatis-plus:
  typeAliasesPackage: com.entity
  global-config:
    sql-injector: com.baomidou.mybatisplus.mapper.LogicSqlInjector

# 新
mybatis-plus:
  type-aliases-package: com.entity
  global-config:
    sql-injector: com.baomidou.mybatisplus.core.injector.LogicSqlInjector
```

#### 6. 启用测试

- 移除 `pom.xml` 中 `<skipTests>true</skipTests>`
- 为关键 service/controller 编写单元测试

### 阶段二：前端升级（Vue 2 → Vue 3）

**预计工作量**: 较大 | **风险**: 中高

#### 1. 核心框架升级

| 旧 | 新 | 变更说明 |
|---|---|---|
| Vue 2.6 | Vue 3.4 | Options API → Composition API (ref/reactive) |
| vue-router 3.1 | vue-router 4 | `new VueRouter()` → `createRouter()`，路由配置语法变化 |
| element-ui 2.13 | element-plus 2.6 | 组件名/属性/事件可能有 breaking changes |
| axios 0.19 | axios 1.7 | 基本兼容，注意拦截器写法 |

#### 2. 构建工具升级

| 旧 | 新 | 说明 |
|---|---|---|
| @vue/cli-service 4.1 | vite 5 + @vitejs/plugin-vue 5 | 告别 webpack，秒级启动 |
| node-sass + sass-loader | sass (Dart Sass) | 直接替换，无兼容问题 |
| babel-eslint | @babel/eslint-parser | ESLint 9 flat config |
| eslint 5 | eslint 9 | 采用 flat config (eslint.config.js) |

#### 3. 主要改动清单

**main.js 改造** (Composition API 入口):
```js
// 旧 (Vue 2)
import Vue from 'vue'
Vue.use(ElementUI)
new Vue({ router, render: h => h(App) }).$mount('#app')

// 新 (Vue 3)
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
const app = createApp(App)
app.use(ElementPlus)
app.mount('#app')
```

**Vue 组件改造** (以 login.vue 为例):
```vue
<!-- 旧 -->
<script>
export default {
  data() { return { rulesForm: {} } },
  methods: { login() { ... } }
}
</script>

<!-- 新 -->
<script setup>
import { ref } from 'vue'
const rulesForm = ref({})
const login = () => { ... }
</script>
```

**vue-amap → 高德地图 JS API 2.0**:
- `vue-amap` 不兼容 Vue 3，需改用原生高德 JS API + ECharts 集成

**vue-quill-editor → quill 2 + @vueup/vue-quill**:
- 或直接替换为 Markdown 编辑器（如 `md-editor-v3`）

**vue-json-excel → vue3-json-excel 或前端生成方案**:
- 可改用 `xlsx` (SheetJS) 库直接生成

#### 4. vue.config.js → vite.config.js

- webpack chain 配置（SVG sprite loader）需改写为 Vite 插件
- 代理配置移至 `vite.config.js` 的 `server.proxy`

## 四、执行顺序（前后端同步升级）

由于前后端通过 REST API 通信，升级时需保持 API 契约一致，建议按以下交错节奏推进：

```
Week 1-2:  后端基建 + 前端基建  (并行)
  ├─ 后端 1.1 升级 JDK 到 17
  ├─ 后端 1.2 升级 Spring Boot 到 3.2
  ├─ 后端 1.3 替换 javax → jakarta
  ├─ 后端 1.4 升级 MyBatis Plus + 修正配置
  ├─ 前端 2.1 脚手架迁移: vue-cli → Vite (保留 Vue 2 代码，先搭好构建)
  └─ 前端 2.2 升级 Vue 3 + vue-router 4 (先升级框架，组件暂不改)

Week 3-4:  后端核心 + 前端核心  (并行)
  ├─ 后端 1.5 升级 Shiro → Spring Security 6
  ├─ 后端 1.6 替换 FastJSON → Jackson
  ├─ 后端 1.7 升级其余依赖 (Hutool, POI, Baidu SDK 等)
  ├─ 后端 1.8 启用测试 + 补充单元测试
  ├─ 前端 2.3 升级 Element UI → Element Plus
  ├─ 前端 2.4 组件迁移: Options API → Composition API (逐页面改造)
  └─ 前端 2.5 构建配置迁移: webpack chain → Vite plugins

Week 5-6:  特殊库替换 + 联调
  ├─ 前端 2.6 特殊库替换: vue-amap / vue-json-excel / vue-quill-editor
  ├─ 前后端联调: 确保所有接口在 Spring Boot 3 + Vue 3 下正常工作
  └─ 修复联调中发现的兼容性问题

Week 7:    加固与优化
  ├─ CI/CD 测试流程启用
  ├─ 安全审计 (依赖漏洞扫描)
  └─ 性能优化 (前端按需加载, 后端连接池调优)
```

### 同步升级的关键注意点

1. **API 契约先行** — 后端升级完成后先确认所有接口返回格式不变（尤其是 JSON 序列化），前端再开始改造
2. **Jackson 替代 FastJSON** — 注意 `@JsonIgnore` / `@JsonProperty` 等注解与 FastJSON 的 `@JSONField` 语法不同
3. **Shiro → Spring Security** — 权限注解从 `@RequiresPermissions` 变为 `@PreAuthorize`，需同步调整前端路由守卫
4. **代理配置对齐** — Vite 的 proxy 配置需与后端 context-path `/gym` 保持一致

## 五、风险评估与缓解

| 风险 | 等级 | 缓解措施 |
|---|---|---|
| Shiro 2.0 API 变动大 | 中 | 评估是否直接迁移至 Spring Security 6 |
| Vue 组件 Options API → Composition API 改造量大 | 高 | 渐进式迁移，先改入口文件再逐个页面 |
| 前端静态样式（内联 style 对象）可能需要调整 | 低 | Element Plus 的 CSS 变量体系兼容大部分样式 |
| org.json 替换为 Jackson | 低 | 仅 2 处使用，直接替换即可 |
| 数据库 schema 不变 | 无 | 升级不影响数据库结构 |

## 六、验证方式

1. **后端**: `mvn clean package -DskipTests=false` 编译通过；`mvn test` 全部通过；`mvn spring-boot:run` 正常启动，访问 `http://localhost:8080/gym` 接口可响应
2. **前端**: `npm run serve` 开发服务器正常启动；浏览器访问 `http://localhost:8081` 页面渲染正确；登录、CRUD 操作功能正常
3. **端到端**: 管理员登录 → 创建会员 → 报名课程 → 购买商品 → 余额充值，完整业务流程可跑通
